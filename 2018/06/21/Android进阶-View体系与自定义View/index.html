Android进阶-View体系与自定义View

发表于 2018-06-21 | 分类于 Android进阶之光 | 阅读次数:

View 与 ViewGroup

  • View 是 Android 所有控件的基类
  • ViewGroup 可以理解成 View 的组合，他可以包含很多 View 或者 ViewGroup
  • ViewGroup 也继承自 View

坐标系

1. Android坐标系

在 Android 中，将屏幕左上角的定点作为 Android 坐标系的原点，这个原点向右是X轴正方向，向下是Y轴正方向，使用 getRawX() 和 getRawY() 方向获得的坐标也是 Android 坐标系的坐标。

2. View坐标系

(1) View 获取自身宽高

(2) View 自身的坐标

  • getTop(): 获取 View 自身顶边到父布局顶边的距离
  • getLeft(): 获取 View 自身左边到父布局左边的距离
  • getRight(): 获取 View 自身右边到父布局左边的距离
  • getBottom(): 获取 View 自身底边到父布局顶边的距

(3) MotionEvent 提供的方法

无论是 View 还是 ViewGroup，最终的点击事件都会由 onTouchEvent(MotionEvent event)方法来处理。它也提供了如下几个常用到的获取焦点坐标的方法

  • getX(): 获取点击事件距离控件左边的距离，即视图坐标。
  • getY(): 获取点击事件距控件顶边的距离，即视图坐标。
  • getRawX(): 获取点击事件距离整个屏幕左边的距离，即绝对坐标。
  • getRawY(): 获取点击事件距离整个屏幕顶边的距离，即绝对坐标。

View 的滑动

View 的滑动是 Android 实现自定义控件的基础，同时我们在实际开发中也经常遇到 View 的滑动处理。
其实无论是哪种滑动方式，其基本思想都是类似的：当点击事件传到 View 时，系统几下触摸点的坐标，手指移动时算出偏移量，并通过偏移量来修改 View 的坐标。实现 View 滑动有多种方法，这里主要讲6种滑动方法，分别是 layout()、offsetLeftAndRight()与offsetTopAndBottom()、LayoutParams、动画、scrollTo 与 scrollBy，以及 Scroller。

3. LayoutParams(改变布局参数)

LayoutParams 主要保存了一个 View 的布局参数，因此我们可以通过 setLayoutParams(newLayoutParams) 来重新布局

4. 动画

可以采用 View 动画来移动，在 res 目录新建 anim 文件夹并创建动画的 xml 文件
注意 View 动画并不能改变 View 的位置参数。在 Android3.0 时出现的属性动画解决了 View 动画结束后位置并没有改变这一问题，也就是说属性动画不仅可以执行动画，还能改变 View 的位置参数。

5. scrollTo 与 scrollBy

scrollTo(x,y) 表示移动到一个具体的坐标点
scrollBy(dx,dy) 表示移动的增量为 dx、dy

6. Scroller

我们在用 scrollTo/scrollBy 方法进行滑动时，这个过程是瞬间完成的，用户体验不太好，我们可以使用 Scroller 来实现有过度效果的滑动，这个过程不是瞬间完成的，而是在一定时间内完成。Scroller 本身是不能实现 View 滑动的，它需要与 View 的 computeScroll()方法配合才能实现弹性滑动的效果

属性动画

在属性动画出现之前，Android 系统只提供了帧动画和 View 动画。View 动画提供了 AlphaAnimation、RotateAnimation、TranslateAnimation、ScaleAnimation 这 4 种动画方式，并且提供可 AnimationSet 动画集合来混合使用多种动画。
随着 Android 3.0 属性动画的退出，View 动画不在风光。相比属性动画，View 动画一个非常大的缺陷突显，其不具有交互性。当某个元素发生 View 动画后，其响应事件的位置依然在动画进行前的原始地方，所以 View 动画只能做普通的动画效果，要避免涉及交互操作。
属性动画优点也特别明显：效率比较高、使用方便。
在Animator框架中使用最多的就是 AnimatorSet 和 ObjectAnimator 配合：使用 ObjectAnimator 进行更精细化的控制，控制一个对象和一个属性值，而是用多个 ObjectAnimator 组合到 AnimatorSet 形成一个动画。属性动画通过调用属性 get、set 方法来真实的控制一个 View 的属性值。

1. ObjectAnimator

ObjectAnimator 是属性动画中最重要的类，创建一个 ObjectAnimator 只需要通过其静态工厂类直接返还一个 ObjectAnimator 对象。参数包括一个对象和对象的属性名字，但这个属性必须有 get 和 set 方法，其内部会通过 Java 反射机制来调用 set 方法修改对象的属性值。

从上看出第一个参数是要操作的 Object ；第二个参数是要操作的属性； 最后一个参数是一个可变的 float 类型数组，需要穿进去该属性变化的取值过程，这里设置了一个参数，变化到200。与 View 动画一样，也可以给属性动画设置显示市场、插值器等属性。
常用的可以直接使用的属性动画的属性值：

  • translationX 和 translationY：用来沿着 X 轴或者 Y 轴进行平移。
  • rotation、rotationX、rotationY：用来围绕 View 的支点进行旋转。
  • PrivotX 和 PrivotY：控制 View对象的支点位置，围绕这个支点进行旋转和缩放变换处理。默认该支点位置就是 View 对象的中心点。
  • alpha：透明度，默认是 1（完全不透明），0 代表完全透明。
  • x 和 y：描述 View 对象在其容器中的最终位置。

2. ValueAnimator

ValueAnimator 不提供任何动画效果，它更像一个数值发生器，用来产生有一定规律的数字，从而让调用者控制动画的实现过程。通常情况下，在 ValueAnimator 的 AnimatorUpdateListener 中监听数值的变化，从而完成动画的变换。

3. 动画的监听

完整的动画具有 start、Repeat、End、Cancel 这 4 个过程。

4. 组合动画—AnimatorSet

AnimatorSet 类提供可一个 play()方法，如果我们向这个方法中传入一个 Animator 对象（ValueAnimator 或者 ObjectAnimator），将会返回一个 AnimatorSet.Builder 的实例。
从 AnimatorSet.class 的源码中看出 AnimatorSet.Builder 类采用可建造者模式，每次调用方法时都返回 Builder 自身用于继续构建。 Animato.Builder 中包括以下 4 个方法。

  • after(Animator anim)：将现有动画插入到传入的动画之后执行
  • after(long delay)：将现有动画延迟指定毫秒后执行
  • before(Animator anim)：将现有动画插入到传入动画之前执行
  • with(Animator anim)：将现有动画和传入动画同时执行

5. 组合动画—PropertyValuesHolder

除了使用 AnimatorSet 类，还可以使用 PropertyValuesHolder 类来实现组合动画，不过这种形式的组合动画没有 AnimatorSet 组合动画实现丰富，使用 PropertyValuesHolder 类只能是多个动画一起执行。

View 的事件分发机制

1. 源码解析 Activity 的构成

点击事件用 MotionEvent 来表示，当一个点击事件产生后，事件最先传递给 Activity ，我们可以先了解一下 Activity 的构成。
当我们写 Activity 时会调用 setContentView() 方法来加载布局。查看 setContentView() 实现
然后调用了 getWindow().setContentView(layoutResID)。查看 getWindow() 实现
发现直接返回了 mWindow 对象
那么这个 mWindow 对象又是什么呢？继续查看源码在 Activity 的 attach() 方法中发现了 mWindow如下：
mWindow = new PhoneWindow(this);
所以 mWindow 指的就是 PhoneWindow，它是继承抽象类 Window 的，这样就知道了 getWindow()得到的其实是一个 PhoneWindow，因为 Activity 中 setContentView() 方法调用的是 getWindow().setContentView(layoutResID)。
在 PhoneWindow 源码中我们找到 setContentView()方法,看到系统会先创建一个 DecorView，这个 DecorView 也就是 Activity 中的根 View。DecorView 也是 PhoneWindow 类的内部类，并且继承了 FrameLayout。DecorView 又将屏幕划分为两个区域 ：一个是 TitleView，另一个是 ContentView，而我们平时做应用所写的布局正式展示在 ContentView 中的。

2. 源码解析 View 的事件分发机制

当我们点击屏幕时，就产生了点击事件，这个事件被封装成了一个类：MotionEvent。 而当这个 MotionEvent 产生后，系统就会将这个 MotionEvent 传递给 View 的层级，MotionEvent 在View 中的层级传递过程就是点击事件分发。 事件分发有3个重要的方法：

  • dispatchTouchEvent(MontionEvent ev) —- 用来进行事件的分发
  • onInterceptTouchEvent(MontionEvent ev) —- 用来进行事件的拦截，在dispatchTouchEvent()中调用，需要注意的是 View 没有提供该方法。
  • onTouchEvent(MotionEvent ev) —- 用来处理点击事件，在dispatchTouchEvent()方法中进行调用
